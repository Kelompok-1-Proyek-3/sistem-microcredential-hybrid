## 1. Executive Summary

Website Group bertanggung jawab untuk **core platform** Microcredential IMPC. Modul eLearning adalah **critical path** yang mengintegrasikan seluruh workflow (enrollment, content delivery, assessment, sesi hybrid offline, scan check-in, certificate generation, external verification). eCommerce mendukung B2C catalog exposure dan checkout experience.

**Deliverables** (Custom Dev — Kustomasi Signifikan):
- Redeem voucher/code validation & generation engine
- Exam gating logic berdasarkan status hadir Events (hybrid only)
- Certificate QR Code + Hash ID generation
- External certificate verification API endpoint

**Deliverables** (Kustomasi Minor — Config + Minor Code):
- Custom field `learning_mode` pada eLearning course
- Linking course ke Events event (FK, read-only referral)
- Display Events e-ticket di eLearning portal dashboard peserta
- Attendance sync dari Events → exam gating trigger
- eCommerce → eLearning integration (catalog + enrollment trigger)

**Native Odoo 19 (Zero Customization Required)**:
- Content management, progress tracking, quiz & scoring via `slide.slide` dan `slide.slide.partner`
- Auto-enrollment on payment — modul `website_sale_slides` (native) menggunakan `slide.channel.enroll='payment'`; enrollment trigger terjadi di `sale.order._action_confirm()`, BUKAN via webhook/API dari Accounting
- Badge portal peserta

> ⚠️ **KOREKSI ODOO 19**: Certificate PDF bukan fitur native `website_slides` (eLearning). Odoo 19 base `website_slides` TIDAK memiliki PDF certificate generation. Sertifikat berbasis survey tersedia via modul `website_slides_survey` + `survey.survey` dengan `certification=True`. Sertifikat custom dengan QR Code & Hash ID (SKENARIO 4) harus dibangun sepenuhnya secara custom.

#### User Story 4B: Sinkronisasi Attendance Events → Exam Gating

**Given**: Admin Marketing Group telah melakukan QR scan check-in peserta via Odoo Mobile App di Events  
**When**: Status `event.registration` berubah menjadi `checkin`  
**Then**: Sistem Website memperbarui status `elearning.session_attendance` untuk membuka akses ujian

> **Catatan Odoo 19**: QR scan check-in fisik dilakukan oleh Marketing Group menggunakan Events + Odoo Mobile App (native, STANDAR). Website Group TIDAK membuat endpoint scan baru. Website Group hanya membaca hasil check-in dari Events dan menyinkronkannya ke tabel attendance eLearning untuk keperluan exam gating.

**Acceptance Criteria**:
- [ ] Scheduled action (setiap 2 menit) membaca `event.registration` dengan `state='done'` (artinya sudah check-in/Attended di Odoo 19)
- [ ] Update `slide.session_attendance.attendance_status = 'HADIR'` jika peserta sudah check-in di Events
- [ ] Lookup berdasarkan: `event.registration.partner_id` → match ke `slide.channel.partner.partner_id`
- [ ] Jika sesi event berakhir dan peserta belum check-in, set `attendance_status = 'TIDAK_HADIR'` otomatis
- [ ] Semua perubahan status tercatat di audit log dengan timestamp
- [ ] Idempotent: Scheduled action aman dijalankan berulang kali tanpa duplikasi

**Database Schema**:
```
# Model custom baru: slide.session_attendance (nama sesuai konvensi Odoo, module prefix slide)
slide.session_attendance:
  - id (primary)
  - channel_partner_id (Many2one ke slide.channel.partner)  # enrollment record
  - event_id (Many2one ke event.event)  # reference ke Events event
  - event_registration_id (Many2one ke event.registration, nullable)  # native Events
  - attendance_status (selection: 'PENDING', 'HADIR', 'TIDAK_HADIR')
  - synced_from_events (bool, default False)  # True jika dari Events sync
  - checkin_time (datetime, nullable)  # diambil dari event.registration.date_closed
  - manual_override (bool, default False)
  - override_by_id (Many2one ke res.users, nullable)
  - create_date (datetime)  # gunakan Odoo native create_date
  - write_date (datetime)  # gunakan Odoo native write_date
  
  _sql_constraints = [('unique_enrollment_event', 'UNIQUE(channel_partner_id, event_id)', 
                       'One attendance record per enrollment per event')]
```

> ⚠️ **KOREKSI ODOO 19**: 
> - `elearning.session_attendance` diganti dengan `slide.session_attendance` (konvensi prefix module)
> - `enrollment_id` diganti dengan `channel_partner_id` (FK ke `slide.channel.partner`)
> - Field `checkin_time` diambil dari `event.registration.date_closed` (Odoo 19 native field untuk attended date)
> - `created_at`/`updated_at` tidak perlu custom: Odoo native menggunakan `create_date`/`write_date`

**Sync Logic** (Scheduled Action setiap 2 menit):
```python
@api.model
def _sync_attendance_from_events(self):
    """Read Events check-in status and update eLearning attendance for exam gating."""
    # Cari HYBRID courses yang punya event
    hybrid_courses = self.env['slide.channel'].search(
        [('learning_mode', '=', 'HYBRID'), ('event_id', '!=', False)]
    )
    for course in hybrid_courses:
        event_regs = self.env['event.registration'].search([
            ('event_id', '=', course.event_id.id),
            ('state', 'in', ['done'])  # 'done' = checked-in di Events native (Odoo 19)
        ])
        for reg in event_regs:
            enrollment = self.env['slide.channel.partner'].search([
                ('channel_id', '=', course.id),
                ('partner_id', '=', reg.partner_id.id)
            ], limit=1)
            if enrollment:
                attendance = self.search([
                    ('channel_partner_id', '=', enrollment.id),
                    ('event_id', '=', course.event_id.id)
                ], limit=1)
                vals = {
                    'attendance_status': 'HADIR',
                    'event_registration_id': reg.id,
                    'checkin_time': reg.date_closed or fields.Datetime.now(),
                    'synced_from_events': True
                }
                if attendance:
                    attendance.write(vals)
                else:
                    self.create({**vals, 'channel_partner_id': enrollment.id, 'event_id': course.event_id.id})
```

- [ ] Jika belum hadir, tampilkan pesan gating + instruksi sesi offline

- [ ] Semua keputusan gating tercatat di log

**Validation Logic**:
```python
def can_open_final_exam(self, partner_id):
    # self = slide.channel record
    self.ensure_one()
    enrollment = self.env['slide.channel.partner'].search([
        ('channel_id', '=', self.id),
        ('partner_id', '=', partner_id),
    ], limit=1)
    if not enrollment:
        return False
    if self.learning_mode == 'ONLINE':
        return True
    if self.learning_mode == 'HYBRID':
        attendance = self.env['slide.session_attendance'].search([
            ('channel_partner_id', '=', enrollment.id),
            ('attendance_status', '=', 'HADIR')
        ], limit=1)
        return bool(attendance)
    return True
```

**When**: Student reaches completion_percentage=100% AND quiz_passed=true AND (if hybrid) attendance_status='HADIR'  
**Then**: Course marked as completed, eligible for certificate

**Acceptance Criteria**:
- [ ] Completion trigger: Auto-detect when conditions met
- [ ] Blocking rule: Jika `slide.channel.partner.completion < 100` ATAU quiz belum lulus ATAU (hybrid dan belum hadir), block certificate generation

**Validation Logic** (Odoo constraint):
```python
# di custom addon, extend slide.channel.partner
@api.constrains('completion', 'member_status')
def _check_completion_eligibility(self):
    for record in self:
        if record.completion == 100 and record.channel_id.learning_mode == 'HYBRID':
            attendance = self.env['slide.session_attendance'].search([
                ('channel_partner_id', '=', record.id),
                ('attendance_status', '=', 'HADIR'),
            ], limit=1)
            if not attendance:
                raise ValidationError("Course hybrid mensyaratkan check-in offline sebelum completion")
```
