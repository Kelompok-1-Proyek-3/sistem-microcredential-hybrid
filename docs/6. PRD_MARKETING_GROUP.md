# PRD: Marketing Group (Events, Surveys)
**Status**: Standar — Owner Events Check-in Hybrid  
**Owner Group**: Marketing Team  
**Referensi**: BPMN Scenario 1 (B2B Setup + mode hybrid), Scenario 3 (Monitoring Progress + offline check-in)  
**Tanggal**: 2026  
**Version**: 2.0 — Revisi Faktual (Odoo 19)

---

## 1. Executive Summary

Marketing Group berfokus pada **campaign support, learner engagement, dan operasional check-in sesi tatap muka hybrid**. 

> **Prinsip Odoo 19**: Marketing Group memegang tanggung jawab **operasional QR scan check-in** sesi tatap muka hybrid menggunakan **Events + Odoo Mobile App** (STANDAR native, zero custom code). Ini adalah critical path untuk course hybrid — attendance data dari Events menjadi sumber kebenaran yang dibaca Website Group untuk exam gating.

**Deliverables** (semua STANDAR — zero custom code):
- **Events Check-in Operations**: QR scan peserta hybrid via Odoo Mobile App (KRITIS untuk hybrid course)
- **Hybrid Session Communication**: Announcement, reminder, post-session templates
- Event campaigns untuk promosi kursus (opsional)
- Post-course feedback surveys (standar)
- Basic marketing analytics
- Konfigurasi integration dengan Website Group (event_id linking ke kursus hybrid)

## 3. Feature Requirements (Standard Configuration)

### 3.1 Events Check-in Operations (KRITIS untuk Hybrid Course)

#### User Story 0A: Persiapan Events untuk Sesi Hybrid

**Given**: Course hybrid akan dipublish oleh Website Group  
**When**: Marketing membuat `event.event` untuk sesi tatap muka  
**Then**: Event siap di-link ke kursus hybrid oleh Website Admin

**Acceptance Criteria**:
- [ ] Buat `event.event` di Events module: nama sesi, tanggal, lokasi, kapasitas
- [ ] Set `event.event.is_published = False` (BUKAN `website_published`) — internal event, tidak perlu halaman publik terpisah
  > ⚠️ **KOREKSI ODOO 19**: Field yang benar adalah `is_published` (dari `website.published.multi.mixin`, tersedia saat modul `website_event` terinstall). Field `website_published` TIDAK ADA di Odoo 19.
- [ ] Tambah Attendee Registration fields: nama, email (wajib)
- [ ] Enable QR ticket: `event.event.event_ticket_ids` dengan QR code per peserta
  > ⚠️ **KOREKSI ODOO 19**: Field yang benar adalah `event_ticket_ids` (One2many ke `event.event.ticket`). Field `ticket_ids` TIDAK ADA di Odoo 19 base `event` module.
- [ ] Koordinasikan `event.id` dengan Website Admin (agar bisa di-link ke kursus)
- [ ] Pastikan event dibuat SEBELUM Website Admin mempublish course hybrid

**Konfigurasi Checklist**:
```
Marketing Admin:
  ☐ Events → New Event
  ☐ Isi: Event Name, Date, Location, Capacity
  ☐ Ticket tab → Enable registration confirmation email (native)
  ☐ Catat event.id → share ke Website Admin
  ☐ Uji registration confirmation: pastikan QR muncul di email konfirmasi

Website Admin (koordinasi):
  ☐ Buka form kursus hybrid
  ☐ Pilih event dari dropdown field event_id
  ☐ Publish course (setelah event tersedia)
```

---

#### User Story 0B: QR Scan Check-in via Odoo Mobile App (STANDAR Native)

**Given**: Hari sesi tatap muka tiba, peserta hadir membawa e-ticket  
**When**: Admin Marketing scan QR tiket peserta  
**Then**: Status peserta berubah ke `state='done'` di Events; Website Group reads untuk exam gating

**Acceptance Criteria**:
- [ ] Admin Marketing download Odoo Mobile App (Community atau Enterprise)
- [ ] Login dengan akun yang memiliki akses `event.manager` atau `event.user`
- [ ] Buka Events → [nama event sesi] → Attendees
- [ ] Tap tombol "Scan QR" → scan QR di tiket peserta (email konfirmasi)
- [ ] Status registrasi berubah ke `'done'` secara native (zero custom code)
- [ ] Admin dapat mark check-in manual jika QR tidak terbaca
- [ ] Setelah check-in: Website Group scheduled action (2 menit) otomatis membaca `state='done'` → update `elearning.session_attendance`

**Setup Checklist untuk Check-in**:
```
Pre-event (Marketing):
  ☐ Install Odoo Mobile App di perangkat admin check-in
  ☐ Test login dengan akun event.manager (bukan akun portal)
  ☐ Test scan QR dengan dummy event di staging
  ☐ Siapkan backup: manual check-in via laptop (Events → Attendees → Mark as Attended)

Hari-H:
  ☐ Buka Events → [event sesi hari ini]
  ☐ Scan QR tiket setiap peserta yang hadir
  ☐ Peserta tanpa QR → manual mark via Attendees list
  ☐ Setelah sesi: export daftar hadir (CSV) untuk backup dokumentasi
```

**Catatan Integrasi**:
- Tidak ada custom code di Marketing Group
- Website Group scheduled action membaca `event.registration.state='done'` setiap 2 menit
  > `state='done'` = peserta sudah hadir/check-in (Attended) di Odoo 19 Events native
- HR Group membaca `slide.session_attendance` (hasil sync Website) untuk monitoring dashboard

### 3.2 Surveys for Course Feedback

#### User Story 3: Post-Course Satisfaction Survey

**Given**: Student completed a course  
**When**: Course completion event occurs  
**Then**: Student receives feedback survey link

**Acceptance Criteria**:
- [ ] Survey template created with key sections:
  - Course content quality (1-5)
  - Instructor quality (if applicable) (1-5)
  - Platform usability (1-5)
  - Difficulty appropriateness (1-5)
  - Open feedback comments
  - NPS question: "How likely to recommend this course? (0-10)"
- [ ] Trigger mechanism: Email survey link sent post-completion (manual or scheduled)
- [ ] Anonymous option: Survey can be configured anonymous/non-anonymous
- [ ] Completion incentive: Optional badge/coupon on survey completion
- [ ] Response deadline: Survey closes after 14 days
- [ ] Dashboard: View response rates and average scores

**Configuration Only**:
- Survey templates in Odoo Surveys
- Email automation in Odoo Marketing/Automations (if installed)

---

#### User Story 4: Survey Analytics & Reporting

**Given**: Survey responses collected  
**When**: Marketing reviews survey dashboard  
**Then**: Insights and actionable data available

**Acceptance Criteria**:
- [ ] Metrics displayed:
  - Response rate (%)
  - Average satisfaction score
  - NPS score
  - Top positive themes (manual analysis)
  - Top complaint themes (manual analysis)
- [ ] Filters: By course, date, audience type (B2C/B2B)
- [ ] Export: CSV/PDF report export
- [ ] Monthly report: Dashboard snapshot for management

**NPS Calculation**:
- Promoters: score 9-10
- Passives: score 7-8
- Detractors: score 0-6
- NPS = %Promoters - %Detractors

## 4. Integration Points

### 4.1 Marketing ↔ Website Integration

**Use Case**: Event/survey links directing to course catalog

```
┌───────────────────────────────┐
│ Events / Survey Campaign      │
│ (Marketing Group)             │
└────────┬──────────────────────┘
         │ UTM-tagged links
         ▼
┌───────────────────────────────┐
│ eCommerce Course Catalog      │
│ (Website Group)               │
└────────┬──────────────────────┘
         │ User enrolls (B2C/B2B)
         ▼
┌───────────────────────────────┐
│ Enrollment Data               │
│ (Website + Accounting)        │
└───────────────────────────────┘
```

**Boundary Note**: Marketing does not modify course pages; only links campaigns to existing pages.

**Boundary Note (Hybrid)**: Marketing menyediakan `event.event` untuk check-in hybrid dan melakukan operasional scan. Attendance data yang terkumpul (`event.registration.state='done'`) DIBACA oleh Website Group untuk exam gating — bukan sebaliknya.

---

### 4.2 Marketing ↔ Website Integration: Events Dependency

**Use Case**: Hybrid course memerlukan `event.event` yang dibuat Marketing sebelum dipublish

```
┌──────────────────────────────────────────┐
│  Marketing Group                         │
│  Membuat event.event                     │
│  Shares event.id ke Website Admin        │
└────────┬─────────────────────────────────┘
         │ Website Admin pilih event_id di form kursus
         ▼
┌──────────────────────────────────────────┐
│  elearning.course.event_id (FK)          │
│  (Website Group — Minor config)          │
└────────┬─────────────────────────────────┘
         │ Hari-H: Marketing scan QR via Odoo Mobile App
         │ event.registration.state → 'done'
         ▼
┌──────────────────────────────────────────┐
│  Website scheduled action (2 menit)      │
│  Update elearning.session_attendance     │
│  attendance_status → 'HADIR'             │
└────────┬─────────────────────────────────┘
         │ Exam gating check
         ▼
┌──────────────────────────────────────────┐
│  Ujian akhir dibuka / ditolak            │
│  (Website Group logic)                   │
└──────────────────────────────────────────┘
```

**Workflow Koordinasi Marketing ↔ Website**:
1. Marketing membuat `event.event` → note `event.id`
2. Marketing share `event.id` ke Website Admin
3. Website Admin link `event_id` pada form kursus hybrid
4. Course hybrid dipublish (event sudah tersedia)
5. Peserta daftar → Events native e-ticket dikirim otomatis
6. Hari-H: Marketing scan QR via Odoo Mobile App
7. Website reads attendance → unlock exam jika sudah hadir

## 7. Dependencies & Risks

| Risk | Probability | Impact | Mitigation |
|---|---|---|---|
