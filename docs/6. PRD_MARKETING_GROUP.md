# PRD: Marketing Group (Events, Surveys)
**Status**: Tanpa Kustomasi Signifikan (Standar)  
**Owner Group**: Marketing Team  
**Referensi**: BPMN Scenario 1-4 (Supportive Process) + Update Hybrid/Check-in (Skenario 1 & 3)  
**Tanggal**: 2026  
**Version**: 1.0

---

## 1. Executive Summary

Marketing Group berfokus pada **campaign support dan learner engagement** menggunakan modul Events dan Surveys. Berdasarkan analisis BPMN update hybrid, proses inti platform (enrollment, payment, learning, check-in, certification) tetap dimiliki Website/HR; Marketing berperan pada komunikasi jadwal sesi tatap muka, reminder kehadiran, dan feedback pasca sesi tanpa kustomisasi kode inti.

**Deliverables**:
- Event campaigns for course promotion (opsional)
- Hybrid session communication packs (announcement, reminder, post-session)
- Post-course feedback surveys (standar)
- Basic marketing analytics
- Integration via configuration (no code customization)

---

## 3. Feature Requirements (Standard Configuration)

### 3.1 Events for Course Promotion

#### User Story 0: Hybrid Session Communication (No Code)

**Given**: Website mempublikasikan jadwal sesi offline untuk course hybrid  
**When**: Marketing menjalankan kampanye komunikasi  
**Then**: Peserta menerima pengumuman lokasi, jadwal, dan instruksi membawa e-ticket

**Acceptance Criteria**:
- [ ] Template email/WA reminder untuk H-7, H-1, H-2 jam
- [ ] Konten wajib: tanggal, jam, lokasi, catatan check-in QR, contact person
- [ ] Segmentasi audience berdasarkan peserta hybrid saja
- [ ] Follow-up template pasca sesi untuk survei kepuasan onsite
- [ ] Tidak ada perubahan logika scan/check-in di modul Marketing

---

#### User Story 1: Create Promotional Webinar Event

**Given**: Marketing wants to promote a new course  
**When**: Marketing creates webinar event in Events module  
**Then**: Event registration page available to target audience

**Acceptance Criteria**:
- [ ] Event creation: Name, description, date/time, speaker, target audience
- [ ] Registration page: Public page with registration form
- [ ] Capacity control: Max participant settings
- [ ] Reminder emails: Auto-send reminders (H-7, H-1, H-1 hour)
- [ ] Event tags: Link event with course category tags
- [ ] Attendance tracking: Mark attendees after event
- [ ] Follow-up: Send course recommendation email post-event

**Configuration Only**:
- No code changes
- Use native Odoo Events form and email automation

---

#### User Story 2: Event to Course Conversion Tracking

**Given**: Event participants attended webinar  
**When**: Some participants enroll in course after event  
**Then**: Marketing can track conversion rate

**Acceptance Criteria**:
- [ ] Participant list export from Events module
- [ ] Match participant emails with course enrollment records (manual or report)
- [ ] Conversion metric: (Participants enrolled / Total attendees) × 100%
- [ ] Dashboard report: Event-wise conversion summary
- [ ] Report frequency: Weekly/monthly reporting

**Note**: If automatic tracking needed in future, can be phase 2 enhancement. For now, manual report sufficient.

---

### 3.2 Surveys for Course Feedback

#### User Story 3: Post-Course Satisfaction Survey

**Given**: Student completed a course  
**When**: Course completion event occurs  
**Then**: Student receives feedback survey link

**Acceptance Criteria**:
- [ ] Survey template created with key sections:
  - Course content quality (1-5)
  - Instructor quality (if applicable) (1-5)
  - Platform usability (1-5)
  - Difficulty appropriateness (1-5)
  - Open feedback comments
  - NPS question: "How likely to recommend this course? (0-10)"
- [ ] Trigger mechanism: Email survey link sent post-completion (manual or scheduled)
- [ ] Anonymous option: Survey can be configured anonymous/non-anonymous
- [ ] Completion incentive: Optional badge/coupon on survey completion
- [ ] Response deadline: Survey closes after 14 days
- [ ] Dashboard: View response rates and average scores

**Configuration Only**:
- Survey templates in Odoo Surveys
- Email automation in Odoo Marketing/Automations (if installed)

---

#### User Story 4: Survey Analytics & Reporting

**Given**: Survey responses collected  
**When**: Marketing reviews survey dashboard  
**Then**: Insights and actionable data available

**Acceptance Criteria**:
- [ ] Metrics displayed:
  - Response rate (%)
  - Average satisfaction score
  - NPS score
  - Top positive themes (manual analysis)
  - Top complaint themes (manual analysis)
- [ ] Filters: By course, date, audience type (B2C/B2B)
- [ ] Export: CSV/PDF report export
- [ ] Monthly report: Dashboard snapshot for management

**NPS Calculation**:
- Promoters: score 9-10
- Passives: score 7-8
- Detractors: score 0-6
- NPS = %Promoters - %Detractors

### 3.3 Campaign Support (Optional)

#### User Story 5: Campaign Landing to Course Catalog

**Given**: Marketing launches campaign (email/social)  
**When**: User clicks campaign link  
**Then**: User lands on relevant eCommerce course catalog page

**Acceptance Criteria**:
- [ ] Campaign links: UTM-tagged links to course pages

- [ ] Tracking: Basic UTM source/medium/campaign tracking in analytics
- [ ] CTA: "Daftar Sekarang" button points to eCommerce checkout
- [ ] Performance metrics: Click-through rate, enrollment conversion

**Configuration Only**:
- No module code change
- Use existing website/eCommerce pages with marketing links

## 4. Integration Points

### 4.1 Marketing ↔ Website Integration

**Use Case**: Event/survey links directing to course catalog

```
┌───────────────────────────────┐
│ Events / Survey Campaign      │
│ (Marketing Group)             │
└────────┬──────────────────────┘
         │ UTM-tagged links
         ▼
┌───────────────────────────────┐
│ eCommerce Course Catalog      │
│ (Website Group)               │
└────────┬──────────────────────┘
         │ User enrolls (B2C/B2B)
         ▼
┌───────────────────────────────┐
│ Enrollment Data               │

```

**Boundary Note**: Marketing does not modify course pages; only links campaigns to existing pages.

## 5. Technical Specifications

### 5.1 Development Approach

| Component | Approach | Notes |
|---|---|---|
| **Events** | Standard Odoo configuration | No custom code |
| **Surveys** | Standard Odoo survey templates | No custom code |
| **Email Automation** | Standard Odoo scheduled actions | Optional |
| **Analytics** | Native reporting + exported analysis | No custom dashboard required |
| **Landing pages** | Reuse existing Website/eCommerce pages | No custom frontend |

### 5.2 Configuration Checklist

- [ ] Create event categories (Webinar, Workshop, Product Launch)
- [ ] Configure event email templates (invitation, reminder, follow-up)
- [ ] Configure hybrid session reminder templates (H-7/H-1/H-2 jam)
- [ ] Create survey templates (post-course, NPS, quality feedback)
- [ ] Configure survey automations (send after course completion)
- [ ] Setup reporting views for event and survey performance
- [ ] Define UTM campaign naming conventions

### 5.3 Security & Compliance

- Survey responses containing personal data must follow privacy policy
- Access control: Marketing can view aggregate survey data, not sensitive learner PII unless authorized
- Event participant data stored securely in Odoo access-controlled records

## 7. Dependencies & Risks

### Dependencies
- **Website module**: Course catalog URLs available for campaigns
- **HR/Website module**: Course completion event list for survey targeting

### Risks
| Risk | Probability | Impact | Mitigation |
|---|---|---|---|
| Low survey response rate | Medium | Medium | Use reminders and incentives |
| Event attendance below target | Medium | Low | Improve targeting and timing |
| Broken campaign links | Low | Medium | Link validation checklist before launch |
| Inaccurate conversion tracking | Medium | Low | Standardize UTM usage |

---

## 8. Non-Functional Requirements

- **Simplicity**: Zero custom code for v1
- **Maintainability**: Use out-of-the-box Odoo features
- **Scalability**: Support 50+ events/year, 100+ survey campaigns/year
- **Usability**: Marketing staff can operate without developer dependency

## 10. Conclusion

Marketing Group **tidak memerlukan kustomasi kode** untuk scope BPMN saat ini. Fokus pada konfigurasi standar Odoo Events & Surveys sudah cukup untuk mendukung business flow. Ini mengurangi risiko konflik development dan mempercepat delivery.

Jika di fase berikutnya diperlukan advanced campaign automation atau attribution analytics, dapat dibuat PRD tambahan sebagai **Phase 2 Enhancement**.
