## 1. Executive Summary

HR Group bertanggung jawab untuk **employee learning tracking dan skills management** dalam platform Microcredential IMPC. Fokus utama adalah monitoring dashboard progres belajar karyawan, skills mapping dari certificate completion, dan learning analytics untuk talent development.

> **Prinsip Odoo 19**: 
> 1. HR Group bekerja **hanya dengan data `hr.employee`** — bukan peserta umum (`res.partner`/Portal User). Peserta umum sepenuhnya dikelola oleh Website Group.
> 2. **Check-in fisik** dilakukan oleh Marketing Group via Events + Odoo Mobile App (STANDAR native). HR Group hanya **membaca status hadir** secara read-only untuk monitoring.
> 3. Skills module (`hr_skills`) berlaku **hanya untuk `hr.employee`**. Tracking kompetensi peserta umum bukan tanggung jawab HR Group.

**Deliverables** (Kustomasi Minor):
- Progress monitoring dashboard (Pivot/Graph View + minor aggregation logic)
- Attendance drill-down per sesi hybrid (read-only dari Events)
- Skills mapping dari course certificate ke `hr.employee.skill_ids`
- Learning analytics & reporting

```sql
    e.id as employee_id,
    e.name as employee_name,
    sc.id as channel_id,
    sc.name as course_name,

    scp.completion as completion_percentage,
```

**Data Sync Mechanism**:
```python
# Scheduled action to sync progress every 5 minutes
# Model: hr.learning_progress (custom)
@api.model
def _sync_learning_progress_from_elearning(self):
    """
    Website Group dan HR Group berada di INSTANCE ODOO YANG SAMA.
    Gunakan direct ORM - BUKAN REST API call.
    """
    # Cari semua karyawan yang punya akun portal
    employees = self.env['hr.employee'].search([('user_id', '!=', False)])
    
    for employee in employees:
        partner = employee.user_id.partner_id
        # Direct ORM ke slide.channel.partner (BUKAN API call)
        enrollments = self.env['slide.channel.partner'].search([
            ('partner_id', '=', partner.id)
        ])
        
        for enrollment in enrollments:
            # Cari attendance untuk course HYBRID
            attendance = self.env['slide.session_attendance'].search([
                ('channel_partner_id', '=', enrollment.id),
            ], limit=1)
            
            hr_progress = self.search([
                ('employee_id', '=', employee.id),
                ('channel_partner_id', '=', enrollment.id),
            ], limit=1)
            
            vals = {
                'completion_percentage': enrollment.completion,
                'member_status': enrollment.member_status,
                'attendance_status': attendance.attendance_status if attendance else 'NOT_REQUIRED',
                'checkin_time': attendance.checkin_time if attendance else False,
            }
            if hr_progress:
                hr_progress.write(vals)
            else:
                self.create({
                    **vals,
                    'employee_id': employee.id,
                    'channel_id': enrollment.channel_id.id,
                    'channel_partner_id': enrollment.id,
                })
    
    # Refresh materialized view
    self.env.cr.execute('REFRESH MATERIALIZED VIEW hr_learning_progress_view')
```

> ⚠️ **KOREKSI ODOO 19**: 
> - `website_api = self.env['res.company']._get_website_api_client()` TIDAK ADA. 
> - Gunakan direct ORM: `self.env['slide.channel.partner'].search([...])`
> - Tidak diperlukan REST API antar modul dalam instance Odoo yang sama.

**Auto-Update Workflow** (Scheduled Action — BUKAN Odoo Bus):
```python
# Di HR Group: scheduled action untuk sync skill dari certificate baru
# ⚠️ KOREKSI ODOO 19:
# - bus.bus BUKAN untuk backend cross-module communication
# - bus.bus adalah untuk real-time FRONTEND WebSocket notification
# - Untuk backend: gunakan ir.cron scheduled action + direct ORM

# HR Group: scheduled action setiap 15 menit
@api.model
def _sync_skills_from_certificates(self):
    """Sync skill updates from new certificates. Employee-only."""
    # Direct ORM ke impc.certificate (BUKAN via bus.bus atau REST API)
    # Website Group dan HR Group berada di instance Odoo yang SAMA
    recent_certs = self.env['impc.certificate'].search([
        ('issued_date', '>=', fields.Datetime.now() - timedelta(minutes=15)),
        ('is_revoked', '=', False)
    ])
    for cert in recent_certs:
        # Cari employee berdasarkan partner_id
        employee = self.env['hr.employee'].search([
            ('user_id.partner_id', '=', cert.partner_id.id)
        ], limit=1)
        if not employee:
            continue  # Peserta bukan employee - skip
        
        # Cari skill mapping untuk course ini
        mappings = self.env['impc.course_skill_mapping'].search([
            ('channel_id', '=', cert.channel_id.id)
        ])
        for mapping in mappings:
            # Cek apakah skill sudah ada
            existing_skill = self.env['hr.employee.skill'].search([
                ('employee_id', '=', employee.id),
                ('skill_id', '=', mapping.skill_id.id),
                ('skill_type_id', '=', mapping.skill_type_id.id),
            ], limit=1)
            
            if existing_skill:
                # Update skill level jika level baru lebih tinggi
                existing_skill.write({
                    'skill_level_id': mapping.skill_level_id.id,
                })
            else:
                # Buat record hr.employee.skill baru via employee_skill_ids
                # BUKAN menulis ke skill_ids (computed read-only field)
                self.env['hr.employee.skill'].create({
                    'employee_id': employee.id,
                    'skill_id': mapping.skill_id.id,
                    'skill_level_id': mapping.skill_level_id.id,
                    'skill_type_id': mapping.skill_type_id.id,
                })
            
            # Catat ke audit log (custom model)
            self.env['hr.employee.skill_mapping_log'].create({
                'employee_id': employee.id,
                'skill_id': mapping.skill_id.id,
                'certificate_id': cert.id,
                'skill_level_after_id': mapping.skill_level_id.id,
                'change_date': cert.issued_date.date() if cert.issued_date else fields.Date.today(),
            })
```

> ⚠️ **KOREKSI ODOO 19 — PENTING**:
> 1. `bus.bus._sendone()` untuk backend cross-module = **SALAH**. `bus.bus` adalah frontend WebSocket saja.
> 2. Gunakan `ir.cron` scheduled action + direct ORM untuk backend komunikasi antar modul.
> 3. `hr.employee.skill_ids` adalah **computed read-only Many2many**. Untuk menambah skill, buat `hr.employee.skill` records via `employee_skill_ids` (One2many) atau langsung `hr.employee.skill`.create(...).
> 4. `elearning.certificate` diganti dengan `impc.certificate` (model custom).
> 5. `elearning.course_skill_mapping` diganti dengan `impc.course_skill_mapping`.

```sql
-- Gunakan nama tabel Odoo 19 yang benar
-- slide_channel_partner = tabel DB dari model slide.channel.partner
-- slide_channel = tabel DB dari model slide.channel
-- impc_certificate = tabel DB dari model impc.certificate
SELECT 
  d.name as department,
  COUNT(DISTINCT e.id) as total_employees,
  COUNT(CASE WHEN scp.completion > 0 THEN 1 END) as learning_engaged,
  AVG(scp.completion) as avg_completion,
  COUNT(DISTINCT cert.id) as certificates_issued,
  DATE_TRUNC('day', scp.write_date)::date as snapshot_date
FROM hr_department d
LEFT JOIN hr_employee e ON e.department_id = d.id
LEFT JOIN res_users u ON u.id = e.user_id
LEFT JOIN res_partner rp ON rp.id = u.partner_id
LEFT JOIN slide_channel_partner scp ON scp.partner_id = rp.id  -- enrollment
LEFT JOIN impc_certificate cert ON cert.channel_partner_id = scp.id
GROUP BY d.id, snapshot_date
```

### 4.1 HR ↔ Website Integration

**Use Case**: Sync learning progress dari eLearning ke HR dashboard

> **Catatan**: `attendance_status` di HR dashboard berasal dari `slide.session_attendance` yang disinkronkan Website Group dari Events. HR Group membaca via Odoo ORM (same DB), bukan REST API call.

```
┌────────────────────────────────────────┐
│  Events Module                           │ (Marketing Group — NATIVE)
│  event.registration check-in            │
└────────┴───────────────────────────────┘
         │ Website syncs (every 2 min)
         ▼
┌────────────────────────────────────────┐
│  slide.session_attendance                │ (Website Group — Custom)
│  + slide.channel.partner                │ (native enrollment)
└────────┬─────────────────────────────────┘
         │ HR reads via Odoo ORM (same DB, every 5 min)
         ▼
┌──────────────────────────────────────────┐
│  hr.learning_progress cache              │ (HR Group — Minor Custom)
│  Materialized view refreshed             │
│  Dashboard updated                       │
└──────────────────────────────────────────┘
```

**Implementasi** (Odoo ORM — bukan REST API):
```python
# HR Group membaca eLearning tables via Odoo ORM (same PostgreSQL DB)
# Lebih efisien dari REST API call; tidak ada HTTP overhead
# KOREKSI: gunakan slide.channel.partner (BUKAN elearning.course_enrollment)
enrollments = self.env['slide.channel.partner'].search([
    ('partner_id', 'in', employee_partner_ids)  # partner dari hr.employee.user_id.partner_id
])
```

### 4.2 HR ↔ Skills Management Integration

**Use Case**: Certificate completion triggers skill update untuk employee

```
┌──────────────────────────────┐
│  Certificate Generated            │ (Website Group)
│  impc.certificate created         │
└────────┴─────────────────────┘
         │ HR reads via direct ORM (BUKAN Odoo Bus)
         ▼
┌──────────────────────────────┐
│  HR Scheduled Action (15 min)     │ (HR Group)
│  Read new impc.certificate via ORM│
│  Match to hr.employee only        │
│  Create hr.employee.skill records  │  # via employee_skill_ids
│  (NOT update skill_ids directly)  │  # skill_ids = computed read-only
└──────────────────────────────┘
```

**Boundary**: Jika `certificate.partner_id` (res.partner) tidak terhubung ke `hr.employee`, skill update di-**skip**. Peserta umum tidak memiliki record `hr.employee`.

## 5. Technical Specifications

### 5.1 Development Stack

| Component | Technology | Notes |
|---|---|---|
| **Data Model** | Odoo HR module (hr.employee, hr.skill) | Extend with learning views |
| **Dashboard** | Odoo Studio / custom Kanban view | Real-time updates |
| **Sync** | Odoo ir.cron (scheduled action) | 5-minute sync interval |
| **Materialized View** | PostgreSQL materialized view | Performance optimization |
| **Charts** | Odoo web charts / Chart.js | Lightweight visualizations |
| **API Client** | requests library | Call Website API |
| **Caching** | Redis (optional) | Cache dashboard data |

### 5.2 Performance Requirements

| Feature | Target | Measurement |
|---|---|---|
| Dashboard load | < 2s | For 1,000 employees |
| Progress sync | < 5 minutes | Scheduled interval |
| Drill-down view | < 1s | Individual employee profile |
| Export (CSV) | < 5s | For 10K records |
| Materialized view refresh | < 30s | Nightly (or hourly) |
| Concurrent dashboard users | 100+ | Peak load test |

### 5.3 Security Considerations

- **Data access**: HR can only see employees in their department (access control)
- **Read-only**: HR cannot modify eLearning records (view only)
- **Notes privacy**: Private HR notes not visible to employees
- **Export security**: Exported PDFs can have watermark "HR Use Only"
- **Audit**: Log all dashboard access & exports
