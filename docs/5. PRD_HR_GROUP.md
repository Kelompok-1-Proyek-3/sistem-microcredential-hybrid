## 1. Executive Summary

HR Group bertanggung jawab untuk **employee learning tracking dan skills management** dalam platform Microcredential IMPC. Fokus utama adalah real-time employee progress monitoring dashboard, attendance monitoring untuk course hybrid, skills mapping dari course completion, dan learning analytics untuk talent development. Kustomasi diperlukan terutama pada dashboard dan skills integration.

**Deliverables**:
- Real-time employee learning progress dashboard
- Real-time hybrid attendance/check-in dashboard
- Skills mapping from certificate completion
- HR learning analytics & reporting
- Integration dengan eLearning progress data

```
hr.learning_progress (computed view from eLearning, synced):
  - id (primary)
  - employee_id (FK to hr.employee)
  - course_id (FK to elearning.course)
  - course_name (varchar, synced from eLearning)
  - learning_mode (selection: 'ONLINE', 'OFFLINE', 'HYBRID')
  - enrollment_id (FK to elearning.course_enrollment)
  - enrollment_date (date)
  - completion_percentage (float 0-100)
  - attendance_status (selection: 'NOT_REQUIRED', 'PENDING', 'HADIR', 'TIDAK_HADIR')
  - checkin_time (datetime, nullable)
  - quiz_score (float, nullable)
  - status (selection: 'IN_PROGRESS', 'COMPLETED', 'AT_RISK')
  - last_accessed_date (datetime)
  - estimated_completion_date (date, calculated)
  - synced_at (datetime)  # Last sync timestamp
  - created_at (datetime)
  
Materialized View (for performance):
  SELECT 
    e.id as employee_id,
    e.name as employee_name,
    c.id as course_id,
    c.name as course_name,
    c.learning_mode,
    sp.completion_percentage,
    CASE
      WHEN c.learning_mode = 'HYBRID' THEN COALESCE(att.attendance_status, 'PENDING')
      ELSE 'NOT_REQUIRED'
    END as attendance_status,
    att.checkin_time,
    sp.quiz_score,
    CASE 
      WHEN sp.completion_percentage = 100 AND sp.quiz_passed THEN 'COMPLETED'
      WHEN sp.completion_percentage < 25 AND ce.enrollment_date < NOW() - interval '30 days' THEN 'AT_RISK'
      ELSE 'IN_PROGRESS'
    END as status,
    sp.last_accessed_date
  FROM hr.employee e
  JOIN elearning.course_enrollment ce ON e.user_id = ce.student_id
  JOIN elearning.course c ON ce.course_id = c.id
  JOIN elearning.student_progress sp ON ce.id = sp.enrollment_id
  LEFT JOIN elearning.session_attendance att ON ce.id = att.enrollment_id
```

**Data Sync Mechanism**:
```python
# Scheduled action to sync progress every 5 minutes
@api.model
def _sync_learning_progress_from_elearning(self):
    """
    Call Website API to get latest student progress data
    Update hr.learning_progress records
    Refresh materialized view
    """
    website_api = self.env['res.company']._get_website_api_client()
    
    # Fetch all enrolled employees' progress
    progress_data = website_api.get('/api/v1/elearning/progress/all')
    
    for record in progress_data['data']:
        hr_progress = self.search([
            ('employee_id', '=', record['employee_id']),
            ('course_id', '=', record['course_id'])
        ])
        
        if hr_progress:
            hr_progress.write({
                'completion_percentage': record['completion_percentage'],
              'attendance_status': record.get('attendance_status', 'NOT_REQUIRED'),
              'checkin_time': record.get('checkin_time'),
                'quiz_score': record['quiz_score'],
                'last_accessed_date': record['last_accessed_date'],
                'synced_at': fields.Datetime.now()
            })
        else:
            self.create({...record, 'synced_at': fields.Datetime.now()})
    
    # Refresh materialized view
    self.env.cr.execute('REFRESH MATERIALIZED VIEW hr_learning_progress_view')
```

- [ ] Audit: Track skill changes with timestamp & course reference

**Database Schema**:
```
elearning.course_skill_mapping:

  - skill_id (FK to hr.skill)

  - id (primary)
  - employee_id (FK to hr.employee)
  - skill_id (FK to hr.skill)

  - skill_level_after (selection)
  - change_date (date)
  - created_at (datetime)
```

**Auto-Update Workflow**:
```python
# In Website Group (eLearning module)
# When certificate generated:

@api.model
def create_certificate_and_update_skills(self, enrollment_id):
    enrollment = elearning.course_enrollment.browse(enrollment_id)
    certificate = self.create({...})
    
    # Call HR API to update skills
    hr_api = self.env['res.company']._get_hr_api_client()
    hr_api.post(f'/api/v1/hr/employees/{enrollment.student_id.id}/skills/update', {
        'certificate_id': certificate.id,
        'course_id': enrollment.course_id.id,
        'certificate_date': certificate.issued_date
    })
    
    return certificate
```

- [ ] Comparison: YoY comparison if applicable
- [ ] Benchmarks: Show vs. company average (if accessible)
- [ ] Export: Export analytics as PDF report

**Database Schema**:
```
hr.learning_analytics_snapshot (materialized view, daily refresh):
  - id (primary)
  - department_id (FK to hr.department)
  - snapshot_date (date)
  - total_employees_in_learning (int)
  - avg_completion_percentage (float)
  - total_certificates_this_period (int)
  - completion_rate (float)
  - created_at (datetime)
```

**Analytics Query**:
```sql
SELECT 
  d.name as department,
  COUNT(DISTINCT e.id) as total_employees,
  COUNT(CASE WHEN sp.completion_percentage > 0 THEN 1 END) as learning_engaged,
  AVG(sp.completion_percentage) as avg_completion,
  COUNT(DISTINCT cert.id) as certificates_issued,
  DATE_TRUNC('day', sp.updated_at)::date as snapshot_date
FROM hr.department d
LEFT JOIN hr.employee e ON e.department_id = d.id
LEFT JOIN elearning.course_enrollment ce ON e.user_id = ce.student_id
LEFT JOIN elearning.student_progress sp ON ce.id = sp.enrollment_id
LEFT JOIN elearning.certificate cert ON sp.enrollment_id = cert.enrollment_id
GROUP BY d.id, snapshot_date
```
