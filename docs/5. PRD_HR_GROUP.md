## 1. Executive Summary

HR Group bertanggung jawab untuk **employee learning tracking dan skills management** dalam platform Microcredential IMPC. Fokus utama adalah monitoring dashboard progres belajar karyawan, skills mapping dari certificate completion, dan learning analytics untuk talent development.

> **Prinsip Odoo 19**: 
> 1. HR Group bekerja **hanya dengan data `hr.employee`** — bukan peserta umum (`res.partner`/Portal User). Peserta umum sepenuhnya dikelola oleh Website Group.
> 2. **Check-in fisik** dilakukan oleh Marketing Group via Events + Odoo Mobile App (STANDAR native). HR Group hanya **membaca status hadir** secara read-only untuk monitoring.
> 3. Skills module (`hr_skills`) berlaku **hanya untuk `hr.employee`**. Tracking kompetensi peserta umum bukan tanggung jawab HR Group.

**Deliverables** (Kustomasi Minor):
- Progress monitoring dashboard (Pivot/Graph View + minor aggregation logic)
- Attendance drill-down per sesi hybrid (read-only dari Events)
- Skills mapping dari course certificate ke `hr.employee.skill_ids`
- Learning analytics & reporting

Materialized View (for performance):
  SELECT 
    e.id as employee_id,
    e.name as employee_name,
    c.id as course_id,
    c.name as course_name,
    c.learning_mode,
    sp.completion_percentage,
    CASE
      WHEN c.learning_mode = 'HYBRID' THEN COALESCE(att.attendance_status, 'PENDING')
      ELSE 'NOT_REQUIRED'
    END as attendance_status,
    att.checkin_time,
    sp.quiz_score,
    CASE

  JOIN elearning.course c ON ce.course_id = c.id

**Data Sync Mechanism**:
```python
# Scheduled action to sync progress every 5 minutes
@api.model
def _sync_learning_progress_from_elearning(self):
    """
    Call Website API to get latest student progress data
    Update hr.learning_progress records
    Refresh materialized view
    """
    website_api = self.env['res.company']._get_website_api_client()
    
    # Fetch all enrolled employees' progress
    progress_data = website_api.get('/api/v1/elearning/progress/all')
    
    for record in progress_data['data']:
        hr_progress = self.search([
            ('employee_id', '=', record['employee_id']),
            ('course_id', '=', record['course_id'])
        ])
        
        if hr_progress:
            hr_progress.write({
                'completion_percentage': record['completion_percentage'],
              'attendance_status': record.get('attendance_status', 'NOT_REQUIRED'),
              'checkin_time': record.get('checkin_time'),
                'quiz_score': record['quiz_score'],
                'last_accessed_date': record['last_accessed_date'],
                'synced_at': fields.Datetime.now()
            })
        else:
            self.create({...record, 'synced_at': fields.Datetime.now()})
    
    # Refresh materialized view
    self.env.cr.execute('REFRESH MATERIALIZED VIEW hr_learning_progress_view')
```

**Auto-Update Workflow** (Odoo Bus / Scheduled Action — bukan direct API call):
```python
# Di Website Group (eLearning module) saat certificate di-generate:
# Kirim event via Odoo bus ke HR Group untuk update skills

# Website Group: trigger Odoo bus event
@api.model
def _after_certificate_generated(self, certificate):
    """Trigger skill update via Odoo bus (no direct API cross-call)."""
    self.env['bus.bus']._sendone(
        'hr.skill.update',  # channel
        'certificate.issued',  # message type
        {
            'certificate_id': certificate.id,
            'course_id': certificate.enrollment_id.course_id.id,
            'partner_id': certificate.student_id.id,  # res.partner
            'issued_date': str(certificate.issued_date),
        }
    )

# HR Group: listen dan update skill (scheduled action setiap 15 menit)
@api.model
def _sync_skills_from_certificates(self):
    """Sync skill updates from new certificates. Employee-only."""
    # Cari employee berdasarkan partner_id (res.partner)
    # PENTING: Hanya update jika partner adalah hr.employee, bukan Portal User biasa
    recent_certs = self.env['elearning.certificate'].search([
        ('issued_date', '>=', fields.Datetime.now() - timedelta(minutes=15)),
        ('is_revoked', '=', False)
    ])
    for cert in recent_certs:
        employee = self.env['hr.employee'].search([
            ('user_id.partner_id', '=', cert.student_id.id)
        ], limit=1)
        if not employee:
            continue  # Peserta bukan employee - skip
        mappings = self.env['elearning.course_skill_mapping'].search([
            ('course_id', '=', cert.enrollment_id.course_id.id)
        ])
        for mapping in mappings:
            self._upsert_employee_skill(
                employee=employee,
                skill_id=mapping.skill_id.id,
                new_level=mapping.skill_level,
                certificate_id=cert.id
            )
```

**Database Schema**:
```
hr.learning_analytics_snapshot (materialized view, daily refresh):
  - id (primary)
  - department_id (FK to hr.department)
  - snapshot_date (date)
  - total_employees_in_learning (int)
  - avg_completion_percentage (float)
  - total_certificates_this_period (int)
  - completion_rate (float)
  - created_at (datetime)
```

**Analytics Query**:
```sql
SELECT 
  d.name as department,
  COUNT(DISTINCT e.id) as total_employees,
  COUNT(CASE WHEN sp.completion_percentage > 0 THEN 1 END) as learning_engaged,
  AVG(sp.completion_percentage) as avg_completion,
  COUNT(DISTINCT cert.id) as certificates_issued,
  DATE_TRUNC('day', sp.updated_at)::date as snapshot_date
FROM hr.department d
LEFT JOIN hr.employee e ON e.department_id = d.id
LEFT JOIN elearning.course_enrollment ce ON e.user_id = ce.student_id
LEFT JOIN elearning.student_progress sp ON ce.id = sp.enrollment_id
LEFT JOIN elearning.certificate cert ON sp.enrollment_id = cert.enrollment_id
GROUP BY d.id, snapshot_date
```
