**Status**: Kustomasi Minor

---

## 1. Executive Summary

Sales Group bertanggung jawab untuk **B2B relationship management dan contract handling** dalam platform Microcredential IMPC. Fokus utama adalah contract lifecycle management, redeem code generation trigger, pengelolaan informasi mode pembelajaran (online/offline/hybrid), dan partner (customer) relationship tracking. Tidak ada kustomasi signifikan yang diperlukan; mostly configuration dan workflow setup.

**Deliverables**:
- B2B contract template & workflow
- Contract metadata untuk mode course dan sesi offline (jika hybrid)
- Redeem code generation triggering
- Partner dashboard for B2B customers
- Integration hooks ke Website & Accounting

## 3. Feature Requirements

### 3.1 B2B Contract Management (SKENARIO 1)

#### User Story 1: B2B Quotation from Contract

**Given**: Sales creates quotation for B2B customer (HR company)  
**When**: Sales defines courses, students count, pricing  
**Then**: Quotation generated with redeem code specifications

**Acceptance Criteria**:
- [ ] Quotation type: Create quotation in sale.order
- [ ] Contract line: Define courses with quantity (# of student slots) and unit price
- [ ] Learning mode visibility: Tampilkan mode course (ONLINE/OFFLINE/HYBRID) pada line item
- [ ] Jika HYBRID: tampilkan ringkasan jadwal & lokasi sesi offline pada quotation lampiran
- [ ] Discount: Support volume discount calculation
- [ ] Validity: Set quote validity date (e.g., 30 days)
- [ ] Contract terms: Define contract period (e.g., 90 days of course access)
- [ ] Redeem code spec: Specify code count, expiry date requirement in line notes
- [ ] Total calculation: Amount = sum of (course_price × quantity) - discount
- [ ] Approval flow: Route for contract review (optional approver)
- [ ] PDF generation: Generate professional quotation PDF

**Database Schema**:
```
sale.order (standard Odoo):
  - contract_type (selection: 'B2C', 'B2B_CONTRACT', default 'B2C')
  - partner_type (selection: 'INDIVIDUAL', 'CORPORATE_HR', 'CORPORATE_OTHER')
  
sale.order.line (extended):
  - course_id (FK to elearning.course, nullable)
  - learning_mode (selection: 'ONLINE', 'OFFLINE', 'HYBRID', synced read-only from course)
  - hybrid_session_summary (text, nullable)
  - redeem_code_count (int, nullable, for B2B lines)
  - redeem_code_expiry_days (int, default 90)
  - student_count (int, for B2B contracts)
  - course_access_duration_days (int, for B2B contracts)
```

**Template/Workflow**:
- Select customer → Select courses (multi-select) → Set quantities → Set contract period → Calculate discount → Generate quote

**Given**: Quotation approved by customer

**Then**: Convert to contract (sale.order → confirmed state)

**Acceptance Criteria**:

- [ ] Untuk course hybrid, email konfirmasi wajib memuat jadwal/lokasi sesi offline dan instruksi check-in

**Contract Status Flow**:
```
DRAFT → QUOTATION → APPROVED → CONFIRMED → ONGOING → COMPLETED → RENEWED/EXPIRED
```

#### User Story 3: Redeem Code Generation Request

**Given**: Contract confirmed  
**When**: Sales system requests redeem codes from Website  
**Then**: Website generates codes and returns them

**Acceptance Criteria**:
- [ ] Trigger: Auto-trigger on contract confirmation OR manual request button
- [ ] API call: Sales group calls Website API `/api/v1/elearning/redeem-codes/generate`
- [ ] Parameters: course_id, quantity, expiry_days, contract_reference
- [ ] Response handling: Receive batch_id, code count, CSV download link
- [ ] Retry logic: If API call fails, allow manual retry (max 3x)
- [ ] CSV download: Auto-download CSV with redeem codes for sending to customer
- [ ] Status update: Update sale.order with `redeem_codes_generated=true`, `redeem_code_batch_id=<batch_id>`
- [ ] Notification: Email CSV attachment to sales person + customer contact
- [ ] Audit: Log redeem code generation request with timestamp & status

**API Integration**:
```
# Sales Group initiates
POST /api/v1/elearning/redeem-codes/generate
{
  "course_id": 5,
  "quantity": 100,
  "contract_id": 12345,
  "contract_reference": "IMPC-B2B-2026-001",
  "expiry_days": 90,
  "requested_by": "sales_manager_1"
}

# Website Group responds
{
  "batch_id": "BATCH-20260423-001",
  "codes_generated": 100,
  "csv_url": "/download/batch-20260423-001.csv",
  "status": "SUCCESS",
  "generated_at": "2026-04-23T10:15:00Z"
}
```

**Database Schema**:
```
sale.order (extended):
  - redeem_codes_generated (bool, default False)
  - redeem_code_batch_id (varchar, FK reference)
  - redeem_codes_requested_at (datetime, nullable)
  - redeem_codes_generated_at (datetime, nullable)
  - redeem_code_batch_status (selection: 'PENDING', 'GENERATED', 'FAILED')
```

### 3.2 CRM & Partner Management

#### User Story 5: HR Partner Account Management

**Given**: New B2B customer (HR company) wants to use platform  
**When**: Sales creates customer account in CRM  
**Then**: HR portal access activated

**Acceptance Criteria**:
- [ ] Customer record: res.partner with company info (name, tax ID, phone, email)
- [ ] Contact records: Multiple HR managers as contacts under company
- [ ] Portal user: Create portal user for HR manager (account.user)
- [ ] Access rights: Limit portal view to their own contracts & redeem codes
- [ ] Contact info: Store HR manager email for redeem code CSV distribution
- [ ] Company profile: Store industry, employee count, training budget (optional fields)
- [ ] Billing address: Separate from shipping (if applicable)

**Database Schema** (Standard Odoo, with extended fields):
```
res.partner (extended):
  - company_type (selection: 'INDIVIDUAL', 'CORPORATE_HR', 'CORPORATE_OTHER')
  - industry_type (selection: 'TECH', 'FINANCE', 'MANUFACTURING', 'OTHER')
  - employee_count (int, nullable)
  - training_budget_annual (float, nullable)
  - contract_start_date (date, nullable)
  
account.user (portal):
  - is_hr_partner_admin (bool, default False)
  - partner_id (FK to res.partner)
```

### 3.3 Contract Lifecycle Workflows

#### User Story 7: Contract Renewal Workflow

**Given**: Contract approaching expiry  
**When**: Sales runs contract review (monthly)  
**Then**: Identify renewal opportunities

**Acceptance Criteria**:
- [ ] Renewal identification: System flags contracts expiring in 30 days
- [ ] Sales task: Create activity/task for sales to contact customer

```
sale.order (extended):

```

## 5. Technical Specifications

### 5.1 Development Stack

| Component | Technology | Notes |
|---|---|---|
| **Data Model** | Odoo sale, crm (standard) | Minimal changes, mostly config |
| **Workflow** | Odoo workflow engine | Native sale.order workflow |
| **API Client** | requests library | Call Website API for redeem codes |
| **Portal** | Odoo portal views | Standard portal framework |
| **Reporting** | Odoo reporting module | SQL-based reports + dashboards |
| **Email** | Odoo mail module | Native email integration |

### 5.2 Configuration Required

- **Sale Order Confirmation**: Workflow automation to call Website API
- **Email Templates**: Contract confirmation, redeem code distribution
- **Portal Customization**: HR manager portal views
- **CRM Stages**: Define B2B sales pipeline stages
- **Quotation Template**: PDF design for B2B contracts
- **Discounts**: Volume discount rules

### 5.3 Security Considerations

- **Code Distribution**: Codes sent to HR manager email (not exposed in portal initially)
- **CSV Export**: Codes visible in CSV but not in UI grid (prevent shoulder-surfing)
- **Portal Access**: Restrict to company-assigned partners (access control enforced)
- **API Key**: Use Odoo API key for Website integration (not exposed to frontend)

## 7. Dependencies & Risks

### Dependencies
- **Website module** ready for redeem code generation API
- **Website module** menyediakan sinkron ringkasan jadwal hybrid per kontrak
- **Accounting module** ready for invoice generation on contract confirmation

### Risks
| Risk | Probability | Impact | Mitigation |
|---|---|---|---|
| Redeem code API timeout | Low | Medium | Implement retry + manual fallback |
| CSV not received by customer | Low | Medium | Enable resend option in portal |
| Contract terms misalignment | Medium | High | Clear documentation + approval workflow |
| Volume discount calculation errors | Low | Medium | Manual review before confirmation |

## 8. Non-Functional Requirements

- **Availability**: Portal should be 99.9% available during business hours
- **Response time**: Contract confirmation < 2s, dashboard load < 3s
- **Scalability**: Support up to 1,000 B2B contracts
- **Auditability**: All contract changes logged with user & timestamp
